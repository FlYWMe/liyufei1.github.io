docker

docker 是一个开源项目，让应用程序部署在软件容器下的工作可以自动化进行。docker在容器基础上进行进一步封装，从文件系统网络互联到进程隔离等，简化了容器的创建和维护。虚拟机是虚拟出一套硬件后在其上运行一个完整的操作系统再在该系统上运行应用进程。而容器内的应用进程直接运行于宿主的内核，容器没有自己的内核，而且也没有进行硬件虚拟，轻便快捷。

  • 镜像 Image

相当于root文件系统，除了提供容器运行时所需的程序、库、资源、配置等文件外，还有为运行准备的配置参数（匿名卷、环境变量、用户等），数据静态，内容构建后不变。

  • 容器 Container

镜像和容器类似于类和实例，容器是镜像运行时的实体，容器可以被创建启动停止删除暂停等。本质是进程，运行于自己独立的命名空间，容器可以拥有自己的root文件系统、网络配置、用户ID空间，容器内的进程运行在隔离的环境里。

  • 仓库 Registry

一个集中的存储分发镜像的服务，在其他服务器上使用当前宿主机上的镜像，一个仓库有多个标签，每个标签对应一个镜像。

Ubuntu镜像，Ubuntu是仓库的名字，包含不同的版本标签如14.04 16.04，可以通过Ubuntu14.04指定具体版本的镜像。

Windows安装docker

下载：docker for Windows

系统要求：64位Windows 10 pro，开启Hyper-V

镜像加速：settings-Daemon

"registry-mirrors": [
"https://registry.docker-cn.com"
  ]

使用镜像

docker pull ubuntu:16.04
docker run -t -i ubuntu:16.04 /bin/bash //-i交互式操作 -t终端
cat /etc/os-release //查看当前系统版本
exit //退出容器

列出镜像
docker image ls

仓库名、标签、镜像ID（镜像的唯一标识）、创建时间、占用空间

删除镜像
docker image rm

可以通过镜像ID、仓库名：标签来删除，先删除标签再触发删除行为。

使用Dockerfile定制镜像

定制镜像：定制每一层添加的配置、文件，可以将每一层的修改、安装、构建、操作命令写入一个脚本Dockerfile来构建、定制镜像。新建Dockerfile

FROM nginx
RUN echo '<h1>Hello,Docker.</h1>' > /usr/share/nginx/html/index.html

构建镜像：在Dockerfile目录执行docker build -t nginx:v3 .

每一条指令建立一层，执行完命令后commit这一层的修改，构成新的镜像。要把需要RUN的指令写在一起。构建命令后面的点：上下文，一般Dockerfile置于空目录下或项目根目录，.上下文目录也不用一定是Dockerfile所在目录，可以 -f ../Dockerfile 指定。

一般情况下使用默认文件名Dockerfile，并将其置于镜像构建上下文目录中。

Dockerfile 命令

CMD：指定容器启动时指定主进程，容器中的应用都以前台执行，没有后台服务的概念。
